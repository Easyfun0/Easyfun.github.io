Vue的起手式

Posted on 2020-05-24

認識前端框架

SPA與框架的意義

🐈SSR(整頁後端產生好)優點:
1.SEO
2.部分JS,CSS
3.降低效能消耗(手機,平板)
🐈SPA優點:
1.過場(換資料用)
2.轉換時間短

🐈活動網站:
1.多頁切換 => SPA
2.少頁不連貫 => SSR
3.有無API => (沒有) SSR
🐈CMS(平台):
1.SEO => SSR
2.時代需要 => 混搭(Hybird)

框架優缺點

🐈優點:
1.程式架構(框架選擇)
2.模組化
3.資源(社群生態)

🐈缺點:
1.照著規則走
2.選擇後的問題(框架掛掉怎麼辦)

🐈為什麼要用框架？
讓畫面與狀態(資料)同步

前端框架與Vue.js趨勢

Angular:最早的框架，需使用typescript
React:門檻較高，接近JS
Vue:開發者為了輕便操作畫面而出

🐈面試題:為什麼Vue.js不用像React一樣自行指定畫面更新?
Vue幫你做了，diff不同，他幫畫面跟資料同步，只要管好資料就行
React單向綁定，資料不可變動
React => diff DOM
Vue => diff Data

🐈Vue.js基本特性與觀念

1.良好的性能
2.平緩的學習曲線
3.模組化與漸進式開發

MVVM:資料與畫面的同步

(1) Object.defineProperty

let a = 1;
let obj = Object.defineProperty({}, "count", {
    get(){
        console.log("get a");
        return a
    },
    set(value){
        a = value;
    }
});

Get
obj.count
Set
obj.count = 3;

先將資料跟畫面綁一起 => 改資料就改畫面

資料與畫面延伸

method(方法)& computed(計算)

1.格式共用/資料處理 => computed
2.需要傳參數/綁定 => methods
3.computed只有在資料變動時才會觸發

    let states = {
    ary: [{ name: "alex" }, { name: "sara" }]
    };
    new Vue({
        el: "#app",
        data: states,
        computed: {
        alex() {
            console.log("computed");
            return this.ary[0];
        }
        },
        methods: {
        getAlex() {
            console.log("methods");
            return this.ary[0];
        }
        }
    });

範例

任務:第二個下拉選單動,連動第三個下拉

1.使用axios取回資料(需要await)
2.迴圈使用v-for
  postal可能陣列或物件
    <option v-for="(city,index) in postal" :value="index">{{ city.name }}</option>
3.寫v-for一律綁key,key是一對一的最好是放資料ID
4.預設參數使用需要特別小心null
5.條件顯示 v-if/v-else(直接消失在DOM裡)
    <h1 v-if="nowArea !== null">{{ nowArea.code }} {{ nowCity.name }} {{ nowArea.name }}</h1>
    <h1 v-else>請選擇縣市區域</h1>
6.computed兩種格式
    (1)function(不能set,單純get)
    (2)物件(可以有get/set功能)
7.reduce(處理函式,初始值,index)
    let ary = ["a", "b", "c", "d"];

    let result = ary.reduce((prev, current) => {
    prev += current
    return prev;
    }, "");

    // 1. "", "a"=>"a"
    // 2. "a", "b"=>"ab"
    // 3. "ab", "c"=>"abc"
    // 4. "abc", "d"=>"abcd"

    console.log(result)
8.連動(watch) => 觀察資料
  建議不能用computed時再使用
    watch:{
        "index.direct":{
        immediate: true, // 預設先做一次
        handler(newVal,oldVal){...}
        }
    }

範例 上傳圖片

1.沒有在vue裡面操作就不需要放入vue(ex:FileReader)
2.File inputs are read only => 改用v-on:change

    <input class="file" type="file" accept="image/*" v-on:change='fileHandler'/>
    <input class="file" type="file" accept="image/*" @change='fileHandler'/>
3.生命週期
  mounted:畫面跟資料結合後，通常用來初始化
4.vue裡取dom使用ref(this.$refs)
5.function功能
    (1)客製化處理資料
    (2)互動
6.排序/交換
    不能使用下面的方法:vue狹持不到,資料會變,畫面不變

        let temp = this.list[index];
        this.list[index] = this.list[index + direct];
        this.list[index + direct] = temp;
    可以改用1: $set

        let temp = this.list[index];
        this.$set(this.list,index, this.list[index + direct])
        this.$set(this.list,index + direct, temp)
    可以改用2: splice

        let target = this.list.splice(index, 1)[0];
        this.list.splice(index + direct, 0, target);

7.Vue只要畫面沒有即時改變,就是錯了
8.splice
9.$set => 強制觸發畫面更新
