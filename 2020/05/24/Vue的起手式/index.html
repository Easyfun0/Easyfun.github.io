Posted on 2020-05-24

認識前端框架

SPA與框架的意義

🐈SSR(整頁後端產生好)優點:
1.SEO
2.部分JS,CSS
3.降低效能消耗(手機,平板)
🐈SPA優點:
1.過場(換資料用)
2.轉換時間短

🐈活動網站:
1.多頁切換 => SPA
2.少頁不連貫 => SSR
3.有無API => (沒有) SSR
🐈CMS(平台):
1.SEO => SSR
2.時代需要 => 混搭(Hybird)

框架優缺點

🐈優點:
1.程式架構(框架選擇)
2.模組化
3.資源(社群生態)

🐈缺點:
1.照著規則走
2.選擇後的問題(框架掛掉怎麼辦)

🐈為什麼要用框架？
讓畫面與狀態(資料)同步

前端框架與Vue.js趨勢

Angular:最早的框架，需使用typescript
React:門檻較高，接近JS
Vue:開發者為了輕便操作畫面而出

🐈面試題:為什麼Vue.js不用像React一樣自行指定畫面更新?
Vue幫你做了，diff不同，他幫畫面跟資料同步，只要管好資料就行
React單向綁定，資料不可變動
React => diff DOM
Vue => diff Data

🐈Vue.js基本特性與觀念

1.良好的性能
2.平緩的學習曲線
3.模組化與漸進式開發

MVVM:資料與畫面的同步

(1) Object.defineProperty

let a = 1;
let obj = Object.defineProperty({}, "count", {
    get(){
        console.log("get a");
        return a
    },
    set(value){
        a = value;
    }
});

Get
obj.count
Set
obj.count = 3;

(2) observable => vue

先將資料跟畫面綁一起 => 改資料就改畫面
<1>生出資料
<2>綁定
<3>改資料
