JavaScript設計模式

Posted on 2019-09-08

設計模式是個也覺問題的方法，任何的程式語言都可應用設計模式的概念，去解決所遇到的問題，主要是降低城市之間的耦合度,提高彈性,擴展性。
設計模式種類很多，介紹其中較常見的為主

🐈 單體模式(Singleton)
🐈 簡單工廠模式(Factory)
🐈 迭代器模式(Iterator)
🐈 裝飾者模式(Decorator)
🐈 策略模式(Strategy)
🐈 外觀模式(Facade)
🐈 代理模式(Proxy)
🐈 中介者模式(Mediator)
🐈 觀察者模式(Observer)

設計模式-小技巧「防止忘記new」

如果忘記new則會針對window產生全域變數，能避免忘記new

範例5-1

忘記new

function CreateNumber(n) {
    this.number = n;
}
var obj = CreateNumber(55);

防止忘記new

function CreateNumber(n) {
    if (this instanceof CreateNumber) {
        this.number = n;
    } else {
        return new CreateNumber(n);
    }
}
var obj = CreateNumber(55);

🐈 JavaScript內建的Array和Object也可不需要new一樣能產生物件

設計模式-單體模式(Singleton)

確保一個類別只有一個物件，並提供對該物件的全域操作

範例5-2

function Cat() {
    if (typeof Cat.instance === "object") {
        return Cat.instance;
    }
    this.name = "Mike";
    Cat.instance = this;
}
Cat.prototype.move = function () {
    return this.name + " move";
};
var cat1 = new Cat();
var cat2 = new Cat();
console.log(cat1 === cat2);
cat1.name = "Phil";
console.log(cat2.move());

設計模式-簡單工廠設計(Factory)

簡單工廠模式是使用一個類別來生成各種不同的物件，由一個方法來決定到底要創建哪個類別的物件，這些物件基本上都擁有相同的介面，可取得需要的物件

範例5-3

Function.prototype.inherits = function (superCtor) {
    Object.setPrototypeOf(this.prototype, superCtor.prototype);
    this.super = superCtor.prototype;
};
function Drink() {

}
Drink.prototype.showColor = function () {
    console.log("Drink color:" + this.color);
}


function Coffee() {
    Drink.apply(this, arguments);
    this.color = "brown";
}
Coffee.inherits(Drink);

function Milk() {
    Drink.apply(this, arguments);
    this.color = "white";
}
Milk.inherits(Drink);

function OrangeJuice() {
    Drink.apply(this, arguments);
        this.color = "orange";
}
OrangeJuice.inherits(Drink);

var factory = {
    build: function (type) {
        return new type();
    }
};
var milk = factory.build(Milk);
milk.showColor();
var coffee = factory.build(Coffee);

設計模式-迭代器模式(Iterator)

迭代器模式是一種提供簡單操作介面而不用暴露來源的方法或更動它。
提供一個操作頁面，讓其他人利用這個界面對資料元素做操作。

範例5-4

var iterator = (function () {
    var index = 0,
        data = [1, 2, 3, 4, 5],
        length = data.length;
    return {
        next: function () {
            index = index + 1;
        },
        hasNext: function () {
            return index < length;
        },
        first: function () {
            index = 0;
        },
        current: function () {
            return data[index];
        }
    }
})();
iterator.first();  //歸零
while (iterator.hasNext()) {
    console.log(iterator.current());
    iterator.next();
}

設計模式-裝飾者模式(Decorator)

裝飾者模式概念是在物件上動態加上功能以強化目標的新增功能，而原本的物件也要被設計成容易強化。原有的方法名稱都沒有改變，但透過複寫方法強化功能，因此功能被稱為裝飾。

範例5-5

function Computer() {
    this.price = function () {
        return 41900;
    }
}
function addMemory(pc) {        //增加
    var price = pc.price();    //覆蓋掉方法     
    pc.price = function () {   
        return price + 10000;
    };
}

function addEngraving(pc) {     //增加
    var price = pc.price();    //取得原本售價
    pc.price = function () {
        return price + 1000;
    };
}
function addInsurance(pc) {     //增加
    var price = pc.price();
    pc.price = function () {
        return price + 8777;
    };
}

var mb = new Computer();

addMemory(mb);
addEngraving(mb);
addInsurance(mb);
console.log(mb.price());

設計模式-策略模式(Strategy)

策略模式可在執行期，依照不同情況選擇不同的策略方式，基本上使用相同的操作介面，但可依照不同的程式來運作

範例5-6

var data = {
    name: "Mike",
    message: "hi hi"
};  //資料封包
function Sender(strategy) {
    this.strategy = strategy;
};  //核心物件，利用建構式置入策略
Sender.prototype.send = function (data) {
    return this.strategy.send(data);
}   //核心方法

var xmlStrategy = {
    send: function (data) {
        var str = "<xml><name>" + data.name + "</name><message>" + data.message + "</message></xml>"
        return str;
    }
}   //訂定策略xmlStrategy
var jsonStrategy = {
    send: function (data) {
        var str = JSON.stringify(data);
        return str;
    }
}
var textStrategy = {
    send: function (data) {
        var str = data.name + ',' + data.message;
        return str;
    }
};  //訂定策略jsonStrategy

var sender = new Sender(textStrategy)
var result = sender.send(data);
console.log(result);

設計模式-外觀模式(Facade)

子系統中的一組介面提供一個統一的高層介面，使得子系統更容易使用，簡化複雜的操作過程

範例5-7

var cpu = {
    excute: function () {
        console.log("excute");
    }
}
var memory = {
    load: function () {
        console.log("load");
    }
}
var hdd = {
    write: function () {
        console.log("write");
    }
}
var computer = {
    work: function () {
        memory.load();
        cpu.excute();
        hdd.write();
    }
}
var user = {
    mainTask: function () {
        computer.work(1);
        computer.work(2);
    }
}
user.mainTask();

設計模式-代理模式(Proxy)

代理模式是用一個物件做為另一個物件的介面，主要目的為保護物件的存取，避免不正確的操作行為

範例5-8

var car = {
    start: function () { console.log("發動"); },
    stop: function () { console.log("關閉"); },
    run: function () { console.log("加速"); }
};
function ProxyCar(car) {
    var isStart = false;
    this.start = function () {
        if (isStart) {
            console.log("已經發動");
        } else {
            car.start();
            isStart = true;    //確保車子已啟動
        }
    };
    this.stop = function () {
        if (!isStart) {
            console.log("已經關閉");
        } else {
            car.stop();
            isStart = false;
        }
    };
    this.run = function () {
        if (!isStart) {
            console.log("還沒發動");
        } else {
            car.run();
        }
    };
}
var ProxyCar = new ProxyCar(car);

設計模式-中介者模式(Mediator)

當物件和物件之間或有錯綜複雜的交互作用，可將這些關係交由另一物件(中介者)來處理，以減少這些物件的耦合

範例5-9

function Player(name) {
    this.points = 0;
    this.name = name;
}
Player.prototype.Player = function () {
    this.points += 1;    //得分+1
    mediator.played();   
}

var scoreboard = {
    el:document.body,
    update: function (score) {
        var msg = "";
        for (var i in score) {
            if (score.hasOwnProperty(i)) {
                mag += "<p><b>" + i + "</b>";
                msg += score[i];
                msg += "</p>";
            }
        }
        this.el.innerHTML = nsg;
    }
}

var mediator = {
    users: {},
    init: function () {  //初始化要做的事
        this.users.home = new Player("Home");
        this.users.guest = new Player("Guest");
    },
    played: function () {
        var score = {
            home: this.users.home.points,
            Guest: this.users.guest.points
        };
        scoreboard.update(score);
    },
    keypress: function (e) {
        if (e.keyCode === 48) {
            mediator.users.home.play();
            return;
        }

範例5-9-2

簡化新增Player時，只需要透過中介者加入新的Player即可

function Player(name) {
    this.points = 0;
    this.name = name;
}
Player.prototype.play = function() {
    this.points += 1;
    mediator.played();
};
var scoreboard = {
    el: document.body,
    update: function(score) {
        var msg = "";
        for (var i in score) {
            if (score.hasOwnProperty(i)) {
                msg += "<p><b>" + i + "</b>:";
                msg += score[i];
                msg += "</p>";
            }
        }
        this.el.innerHTML = msg;
    }
};
var mediator = {
    users: {},
    add: function(key, obj) {
        this.users[obj.name] = {
            key: key,
            obj: obj
        };
    },
    played: function() {
        var score = {};
        for (var user in this.users) {    //物件屬性迴圈
            score[user] = this.users[user].obj.points;
        }
        scoreboard.update(score);
    },
    keypress: function(e) {
        for (var user in mediator.users) {
            user = mediator.users[user];
            if (e.keyCode === user.key) {
                user.obj.play();
                return;
            }
        }
    }
};
mediator.add(48, new Player("Home"));
mediator.add(49, new Player("Guest"));
mediator.add(50, new Player("Other"));
mediator.add(51, new Player("Mark"));
mediator.add(52, new Player("John"));
window.onkeypress = mediator.keypress;

設計模式-觀察者模式(Observer)

觀察者模式又為訂閱者模式，是一對多的關係，讓多個觀察者同時監聽某一個主題物件，當這個主題物件狀態發生改變就會通知觀察者。JavaScript就是屬於觀察者模式。

範例5-10

var messageCenter = {           //觀察者核心
    events: {},
    emit: function (type, message) {
        if (this.events[type]) {
            for (var i = 0; i < this.events[type].length; i++){
                this.events[type][i].callback(message);
            }
        }
    },
    on: function (obj, type, callback) {            //目標,事件類型,callback
        this.events[type] = this.events[type] || [];
        this.events[type].push({
            obj: obj,
            callback: callback
        })
    },
    off: function (obj, type) {
        if (this.events[type]) {
            for (var i = 0; i < this.events[type].length; i++){
                if (this.events[type][i].obj === obj) {
                    this.events[type].splice(i, 1);//splice插入或者移除任意索引數
                    i--;
                }
            }
        }
    }
}

function User(messageCenter) {
    this.messageCenter = messageCenter;
}
User.prototype.addEvent = function (type, callback) {
    this.messageCenter.on(this, type, callback);
}
User.prototype.removeEvent = function (type) {
    this.messageCenter.off(this, type);
}

var UserA = new User(messageCenter);     //messageCenter觸發
UserA.addEvent("todo", function (msg) {
    console.log("UserA todo:" + msg);
})

UserA.addEvent("test", function (msg) {
    console.log("UserA test:" + msg);
})

var UserB = new User(messageCenter);
UserB.addEvent("todo", function (msg) {
    console.log("UserB todo:" + msg);
})

var UserC = new User(messageCenter);
UserC.addEvent("test", function (msg) {
    console.log("UserC test:" + msg);
})

messageCenter.emit("todo", "Easyfun0");
messageCenter.emit("test", "Easyfun");
