JavsScript物件繼承

Posted on 2019-09-04

子層類別會擁有父層類別的所有屬性,方法，再加上自己定義的屬性,方法，所以可說子層類別是父層類別的延伸。由於JavaScript屬於prototype-based的物件導向語言，藉由將現有object做為prototype的方式來達成繼承功能

物件繼承-call,apply,bind比較

call就是呼叫函式，不過可透過call傳入第一個參數去改變this，後續的參數當作一般參數傳入
apply差別在後續參數是一個陣列，bind大致和call相同但會產生新的函式
call,apply,bind都是屬於函式的原型方式(Function.prototype)

範例4-1

使用call呼叫函式可傳入參數並且修改this

function show(a, b) {
    console.log(this, a, b);
}
//show(5,10);
show.call({}, 5, 10);

範例4-2

call,apply,bind比較

function base() {
    console.log(this, arguments);
}
base.call({ name: "Mike" }, 2, 3);
base.apply({ name: "Phil" }, [2, 3]);
base.bind({ name: "Mary" })( 2, 3);

函式名稱.call(改變this 物件, 參數1, 參數2,…) => 執行函式
函式名稱.apply(改變this 物件, 陣列) => 執行函式
函式名稱.bind(改變this 物件, 參數1, 參數2,…) => 新函式

🍋 使用new建立物件，其實也就是等同於:

var a = {};
MyObject,call(a, "Mike", 30);

物件繼承-繼承屬性與方法

繼承物件時父層類別所擁有的屬性與方法，子層類別也必定同時擁有相同屬性與方法

範例4-3

透過call繼承屬性與方法

function A() {
    this.abc = 12;
    this.run = function() {
        console.log("go");
    };
}
A.prototype.show = function() {
    console.log(this.abc);
};
function B() {
    A.call(this);
}
var obj = new B();
console.log(obj.abc);
obj.run();

物件繼承-為什麼call能夠繼承屬性與方法

參數傳遞過程是將變數記憶體位置參考複製到function的參數上，當修改這個參數的內容，實際上是去修改參考上的內容，若重新賦予新值則會建立新的記憶體參考，此稱為共享參考(call by sharing)
但由於共享參考在運作上看起來很像傳值參考(call by value)與傳址參考(call by reference)，差別在於原始型別的值是不可變動(immutable)

範例4-4

共享參考

var arr1 = [1, 2, 3];
var arr2 = arr1;
arr2.push(4);
console.log(arr1);
console.log(arr2);

避免共享參考

var arr1 = [1, 2, 3];
var arr2 = arr1.slice();
arr2.push(4);
console.log(arr1);
console.log(arr2);

🍞因此透過call繼承時是透過共享參考把this傳入父層類別

物件繼承-繼承原型方法

call可繼承屬性與方法，但是無法繼承原型方法，因為原型方法是從prototype設定的，子層物件並非父層類別的原型，所以無法透過call繼承原型方法

範例4-5

透過prototype設定可繼承原型方法

function A() {
    this.abc = 12;
}
A.prototype.show = function() {
    console.log(this.abc);
};
function B() {
    A.call(this);
}
B.prototype = A.prototype;
var obj = new B();
console.log(obj.abc);
obj.show();

🔥 雖然可透過prototype設定原型方法，但別忘記共享參考這件事情

範例4-6

透過prototype設定可繼承方法，但是會影響原有的父層類別的prototype

function A() {
    this.abc = 12;
}
A.prototype.show = function() {
    console.log(this.abc);
};
function B() {
    A.call(this);
}
B.prototype = A.prototype;
var objA = new A();
var objB = new B();
B.prototype.square = function () {
    console.log(this.abc * this.abc);
};
objA.square();
objB.square();

物件繼承-繼承原型方法避免共享傳遞

透過prototype設定一個物件，當找不到方法或屬性時才會去原型裡面去找，所以擴充子層類別的原型方法，就不會影響父層類別

範例4-7

prototpye設定一個物件

function A() {
    this.abc = 12;
}
A.prototype.show = function() {
    console.log(this.abc);
};
function B() {
    A.call(this);
}
B.prototype = new A();
var objA = new A();
var objB = new B();
B.prototype.square = function () {
    console.log(this.abc * this.abc);
};
objA.square();
objB.square();

💥參數傳遞方式:
傳值參考:複製內容到新的記憶體上
傳址參考:複製參考到新的記憶體位置上，但修改或賦予新值都會改變原有參考
共享參考:複製參考到新的記憶體位置上，修改會改變原有參考，但賦予新值則會指向新參考的記憶體位置

JavaScript實做的是共享參考，但執行起來卻很像傳值參考與傳址參考。由於原始型別直視不可變動(immutable)的，所以無法修改，永遠都只會賦予新值，所以看起來很像傳值參考。
物件型別雖然是可變動(mutable)的，但常常忽略掉傳址參考(call by reference)賦予新值會影響改變原有參照的這個原則，所以很容易會被誤解是傳址參考
🥕new這個動作其實就是跟記憶體要求一個空間存放新建立物件，也等同於Object.creat([prototype])和[Class].call(obj)這樣的操作
🥕若要修改物件的prototype則可使用Object.setPrototypeOf這個方法

物件繼承-傳遞參數到父層類別

前一個方式沒有辦法傳遞參數到父層類別，因為使用call沒有帶入傳入參數，可改用apply將arguments傳入父層類別中

範例4-8

傳遞參數到父層類別

function A(abc) {
    this.abc = abc || 12;
    console.log("A class");  //繼承時會執行兩次
}
A.prototype.show = function() {
    console.log(this.abc);
};
function B() {
    A.apply(this, arguments);
}
B.prototype = new A();
var objB = new B(11);
B.prototype.square = function () {
    console.log(this.abc * this.abc);
};
objB.square();

物件繼承-暫時類別繼承方法

使用暫時類別繼承方法，不會讓父類別的建構函式重複執行兩次，利用暫時類別的prototype設定父類別的prototype，子類別prototype再設定暫時類別產生出來的物件。子類別super方法設定父類別的prototype，最後重新設定子類別prototpye.constructor到子類別，否則子類別的constructor會是父類別的constructor

範例4-9

使用暫時類別的繼承方法

Function.prototype.inherits = function (superCtor) {
    function F() {
    };
    F.prototype = superCtor.prototype;
    this.prototype = new F();
    this.super = superCtor.prototype;
    this.prototype.constructor = this;
};
function A(abc) {
    this.abc = abc || 12;
    console.log("A class");
}
A.prototype.show = function() {
    console.log(this.abc);
};
function B() {
    A.apply(this, arguments);
}
B.inherits(A);
var objB = new B();
B.prototype.square = function () {
    console.log(this.abc * this.abc);
};
objB.square();

物件繼承-Object.create繼承方式

Object.create就是建立一個物件，並且同時指定物件的＿proto＿原型鏈，因此可取代使用臨時類別的方式

範例4-10

Object原型繼承方式

Function.prototype.inherits = function (superCtor) {
    this.prototype = Object.create(superCtor.prototype);
    this.super = superCtor.prototype;
    this.prototype.constructor = this;
};
function A(abc) {
    this.abc = abc || 12;
    console.log("A class");
}
A.prototype.show = function() {
    console.log(this.abc);
};
function B() {
    A.apply(this, arguments);
}
B.inherits(A);
var objB = new B();
B.prototype.square = function () {
    console.log(this.abc * this.abc);
};
objB.square();

物件繼承-繼承的實際案例

1.建立Point類別，透過write方法設定x和y座標，透過read方法取得座標物件
2.建立RangePoint類別，繼承Point類別並限制範圍，複寫write方法
3.建立IntegerRangePoint類別，繼承RangePoint類別，複寫write方法並四捨五入

範例4-11

Object原型繼承方法

Function.prototype.inherits = function (superCtor) {
    Object.setPrototypeOf(this.prototype, superCtor.prototype);
    this.super = superCtor.prototype;
};

🌽使用Object.setPrototypeOf可不需要重新設定子類別的prototype.constructor，因為是將目前子類別的prototype中的＿proto＿原型鏈設定為父類別的prototype，因此子類別的constructor仍然還是指向子類別

建立Point類別

function Point() {
    this.x = 0;
    this.y = 0;
}
Point.prototype.write = function (x, y) {
    this.x = x;
    this.y = y;
};
Point.prototype.read = function () {
    return { x: this.x, y: this.y };
};
var p = new Point();
p.write(5, 7);
console.log(p.read());

建立RangePoint類別

function RangePoint(min, max) {
    Point.apply(this, arguments);
    this.min = min;
    this.max = max;
}
RangePoint.inherits(Point);
RangePoint.prototype.write = function (x, y) {
    x = x < this.min ? this.min : x;
    x = x > this.max ? this.max : x;
    y = y < this.min ? this.min : y;
    y = y > this.max ? this.max : y;
    RangePoint.super.write.apply(this, arguments);
};
