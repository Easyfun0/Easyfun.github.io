JavaScript語言特性

Posted on 2019-08-23

JavaScript雖然和Java很類似，但卻沒有 Java 的靜態定型（static typing）及強型態確認（strong type checking）特性，JavaScript變數資料型態沒有宣告就可使用（動態定型，dynamic typing)，我們先前了解了變數範圍後，接下來深入的去理解JavaScript的特性。

閉包(Closure)

閉包的概念為一個函式操作到另一個函式作用範圍內的變數。但只有巢狀結構的函式才會形成閉包，如果內層函式引用到外層函式的區域變數，外層函式中的變數狀態就會被內層函式保留下來(形成封閉狀態，只有內層函式可存取，達到保護變數內容目的)。

範例2-1

var g;
function outer() {
    var a = 100;
    function inner() {
        console.log(a);
    }
    g = inner;
}
outer();
g();

outer是100
g是undefined

範例2-2

使用閉包方法

印出a
var name = "a";
var obj = {
    name: "b",
    getName: function () {
        return function () {
            return this.name;
        };
    }
};
var result = obj.getName();
console.log(result());

印出b的方法
var name = "a";
var obj = {
    name: "b",
    getName: function () {
        var that = this;
        return function () {
            return that.name;
        };
    }
};
var result = obj.getName();
console.log(result());

印出b的方法就是在宣告一個that在外面，裡面的that在使用外面宣告的

範例2-3

利用閉包記憶變數

function add() {
    var count = 0;
    return function () {
        return count += 1;  //keep住變數
    }
}
var user = add();
var custom = add();

當我們要顯示每一秒顯示0~10的數字，先用for迴圈然後使用setTimeout

for(var i = 0; i < 10; i++) {
    setTimeout(function() {
        console.log(i);
    }, i*1000);
}

咦！ 他印10個10耶 那就錯了

範例2-4

我們可以使用閉包解決印出重複地問題

for(var i = 0; i < 10; i++) {
    (function(i) {
        setTimeout(function() {
            console.log(i);
        }, i*1000);
    })(i)
}

Currying

Curring是利用閉包將傳入的變數保存下來，將多個參數的function，轉變接受一個參數的function，簡化傳入參數的一種方式。
好處是提高程式的彈性和可讀性,易於重複利用。

範例2-5

參數重複傳入

function cube(height, width, depth) {
    return height * width * depth;
}
console.log(cube(3, 2, 1));
console.log(cube(3, 2, 2));
console.log(cube(3, 2, 3));
console.log(cube(3, 2, 4));
console.log(cube(3, 2, 5));

範例2-6

將參數利用閉包保存

function cube(height) {
    return function (width) {
        return function (depth) {
            return height * width * depth;
        };
    };
}
console.log(cube(3)(2)(1));

Hoisting

hoisting是指「一種把宣告提升到其所區域內最頂端」的內部行為，但是提升行為僅只針對宣告的部分，對於變數的設定，並沒有跟著提升

範例2-7

將變數宣告提升

var x = 5;
(function () {
    console.log(x);
    var x = 10;
})();
console.log(x);

程式實際執行過程

var x = 5;
(function () {
    var x;
    console.log(x);
    x = 10;
})();
console.log(x);
x為undefined跟外面的5

範例2-8

將變數宣告提升

var a = 1;
function hoisting() {
    if (!a) {
        var a = 2;
    }
    console.log(a);
}
hoisting();

程式實際執行過程

var a = 1;
function hoisting() {
    var a;
    if (!a) {
        a = 2;
    }
    console.log(a);
}
hoisting();

嚴格模式

嚴格模式主要是讓語法變得更嚴僅，相對於原本的語法執行時比較不易出錯，只要加上”use strict”的字串就是嚴格模式，可寫在每個js檔案的最前面，這樣整個js都會套用嚴謹模式，也可針對function設定嚴格模式。

範例2-9

針對個別function設定嚴格模式

function f(x) {
    "use strict";
    var a = 12;
    b = a + x;
}
f(5);

在嚴格模式下會發生錯誤的情況:
● 未宣告使用變數
● 嘗試寫入唯獨屬性
● 對無法擴充的物件新增屬性
● 定義重複屬性
● 使用未來保留字(implements,interface,let,package,private,protected,public,static,yield)
● 八進位
● 使用eval 或arguments 名稱當做變數或函示名稱
● 區塊內宣告function(if,for)
● 使用with,使用在eval語句宣告的變數
● 修改arguments內的值
● 使用arguments.callee (匿名函式)

範例2-10

在嚴格模式下無法使用arguments.callee

(function () {
    "use strict";
    console.log(Date.now());
    setTimeout(arguments.callee, 1000);
})();

可修改成

(function run() {
    "use strict";
    console.log(Date.now());
    setTimeout(run, 1000);
})();

範例2-11

在嚴格模式下區塊內不能定義function

(function () {
    "use strict";
    if (true) {
        function run() {
            console.log("run");
        }
    }
    run();
})();

浮點運算

電腦數字都是由二進位數進行運算，在轉換成十進位顯示。JavaScript的數值採用IEEE 754在標準64位元浮點數，因此對於數字處理有些小數轉換成二進位小數會造成無限循環，導致轉回十進位時造成誤差。
資料來源: Online Binary-Decimal Converter

範例2-12

浮點數運算誤差

console.log(0.1 +0.2 === 0.3);  //false

解決浮點數運算誤差

console.log(parseFloat((0.1 +0.2).toFixed(15)) === 0.3); //true

大括號排版

大括號不要另起新的一行，因為JavaScript會在return後自動加上分號

範例2-13

大括號在同一行(Good)

if (true) {
    console.log("good");
}

大括號在另一行(Bad)

if (true)
{
    console.log("bad");
}

等號=

在使用等號的時候，盡可能不要使用兩等號的判斷==(!=)，進行比較時會自動轉換型別，應使用三等號的判斷===(!==)，避免因為自動轉型造成判斷上的不明確

範例2-14

自動轉型(Bad)

使用==或!= 判斷時，等號兩邊的值會先自動轉型，再進行比較
console.log(0 == ""); // true
console.log(1 == true); // true
console.log(2 == true); // false
console.log(0 == "0"); // true
console.log(false == "false"); // false
console.log(false == "0"); // true
console.log(" \t\r\n " == 0); // true

不自動轉型(Good)

使用===或!== 判斷時，等號兩邊的值不會自動轉型，直接進行比較
console.log(0 === ""); // false
console.log(1 === true); // false
console.log(2 === true); // false
console.log(0 === "0"); // false
console.log(false === "false"); // false
console.log(false === "0"); // false
console.log(" \t\r\n " === 0); // false

語句合併

要將不同的語句合併成一行，又或者是在if內誤寫一個等號

範例2-15

var a,b;
a = b;
if(a) {
    console.log("a");
}
//會合併成這樣(Bad)
if(a = b) {         //b會傳值到a
    console.log("a=b");
}
//但會看成這樣
if(a == b) {
    console.log("a==b");
}

變數宣告

變數宣告一律都放在函式內最前面，可避免Hoisting問題

範例2-16

Hoisting問題

if(!o) {
    var o = {}; //(Bad)
}
宣告一律寫在上面
var o;
if(!o) {
    o = {};
}

全域變數

竟量避免全域變數使用，改使用立即函式，若無法避免，在變數名稱前加上底線作區隔

範例2-17

var upperCase;
//放在立即函式裡面
(function() {
var _upperCase;
})();

命名方式

避免忘記new，建構函式命名字首應為大寫

範例2-17

function myObject() { //(Bad)
this.x = 100;
}
var p = myObject(); //建構函式大寫(Good)
function MyObject() {
this.x = 100;
}
var q = new MyObject();

✪變數命名規則:
底線(_)和($)是變數中唯二可用的符號，其餘必須是字母或數字，無法數字開頭命名，保留字的部分也無法使用(例:switch,case,if,else等)

◉ 變數使用駝峰式命名(Camel Case):
第一個單字首小寫，第二個單字首大寫，例:name,firstName
◉ 類別使用巴斯卡命名(Pascal Case):
每個單字首都為大寫，例:Server,CreatPerson
◉ 常數(固定的值)使用全大寫命名:
每個單字使用底線連接，例:varPI =3.14,var MAX_WIDTH = 800
◉ 私有變數使用底線開頭命名
不公開使用的變數通常前面加上底線，例:_width,_userSetting
◉ 系統內部使用前後雙底線命名
例: ＿proto＿ , ＿defineGetter＿

自增與自減

i++是顯示變數後在進行累加，++i則是先累加後再顯示變數的值

範例2-18

var i = 5;
console.log(i++); //先印再加
console.log(i);  //顯示改變的結果

var i =5;
console.log(++i); //先加在印
console.log(i);

區塊

總是使用大括號，避免因分號導致語意不同

範例2-19

if(true) console.log("a"); console.log("b");
//原意可能是
if(true) {console.log("a"); console.log("b");}
//實際執行是
if(true) {console.log("a");} console.log("b");
