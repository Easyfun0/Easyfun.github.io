JavaScript物件導向

Posted on 2019-08-28

物件導向就是將程式設計的過程擬物化，也是將軟體想像成由一群物件交互合作所組成。每個物件都可視為獨一無二的個體，他們扮演不同的角色並有不同的能力及責任。

類別與物件

類別(Class):可比喻成藍圖或是範本，用來產生物件的基礎
物件(Object):透過類別產生物件，每個物件都是各自獨立

物件導向特性

🌼 封裝:隱藏細節，不需要知道內部原理，只需要知道如和操作即可。同時確保使用者無法任意更改物件內部重要的資料。
🌼 繼承:可從父類別繼承出方法和屬性，子類別漚可擴充出一些自己的特性。透過繼承減少重複功能的程式碼，只需擴充出不同功能的部分即可。
🌼 多行:可依照不同情形變換資料型態，指一個物件可扮演不同角色特性，

物件的組成

「方法」與「屬性」的不同

🍓 方法(屬於一個物件)等同於函式(自由的,不屬於任何)
🍓 屬性(屬於一個物件)等同於變數(自由的,不屬於任何)

物件導向-this

this為當前function執行所在位置的參考

範例3-1

this會指向當前function(方法)屬於ary的物件

var ary =[1,2,3];
ary.a = 12;  //屬性
ary.show = function() {
    console.log(this.a); //函式執行的位置
};
ary.show();

this會指向當前function(方法)屬於document的物件

doucment.onlcick = function() {   //看在哪個物件上執行
    console.log(this);
};

this會指向window

function show() {
    console.log(this);  //function函式不屬於任何
};
show();

等同下面寫法

window.show = function() {
    console.log(this);
};
window.show();

✪ 不建議在內建物件上隨意加入屬性或方法，否則可能會覆蓋內建的屬性與方法
常用JavaScript內建物件

名稱 說明 名稱 說明
object 物件 Number 數字(含浮點數)
Function 函式 Date 日期物件
Array 陣列 RegExp 正規表示式物件
String 字串 Error 錯誤物件

建立物件1

建立一個物件並且增加屬性與方法

範例3-2

var obj = new Object();
obj.name = "john";
obj.gender = "male";
obj.showName = function () {
    console.log("my name:" + this.name);
};
obj.showGender = function () {
    console.log("my gender:" + this.gender);
};
obj.showName();
obj.showGender();

範例3-3

大量產生物件(工廠方式)

function createPerson(name, gender) {
    //原料
    var obj = new Object();
    //加工
    obj.name = name;
    obj.gender = gender;
    obj.showName = function () {
        console.log("my name:" + this.name);
    };
    obj.showGender = function () {
        console.log("my gender:" + this.gender);
    };
    //出廠
    return obj;
}
var p1 = createPerson("Mike", "male");
p1.showName();
p1.showGender();
var p2 = createPerson("Phil", "female");
p2.showName();
p2.showGender();

建立物件2

工廠方式雖然可以解決重複複製貼上程式碼問題，但卻衍生出另外的問題

🐷問題1: 沒有使用new
系統內建物件習慣上燈有使用new產生物件

var date = new Date();
var ary = new Array();

🐷問題2: 方法重複構造
兩個物件的方法不ㄧ樣，表示每次建立一個新物件就會產生一個新的方法，容易造成記憶體資源的浪費

var date1 = new Date();
var date2 = new Date();
console.log(date1.forEach === date2.forEach); //true

物件導向-new

new工作管理就是從類別產生的物件。由於JavaScript沒有class，不過可以使用function當作class

範例3-4

使用new與不使用new的差異

function show() {
    console.log(this);
}
show();
new show();

範例3-5

new產生物件 比較範例3-3工廠方式

function createPerson(name, gender) {
    //系統內部執行
    //var this = new Object();

    //加工
    this.name = name;
    this.gender = gender;
    this.showName = function () {
        console.log("my name:" + this.name);
    };
    this.showGender = function () {
        console.log("my gender:" + this.gender);
    };
    //系統內部執行
    //return this;  不用return
}
var p1 = new createPerson("Mike", "male");
var p2 = new createPerson("Phil", "female");

物件導向-prototype

原型(prototype)可解決建立物件時，重複產生方法的問題。只有在類別上面才可建立原型方法，在物件上無法建立原型方法

範例3-6

替物件加上方法，只有自己的物件有這個方法

var ary1 = new Array(12, 23, 56, 75, 33);
ary1.sum = function () {
        var output = 0;
        for (var i = 0; i < this.length; i++) {
            output += this[i];
        }
        return output;
};
console.log(ary1.sum());   //沒有sum的function
var ary2 = new Array(12, 33, 55);
ary2.sum();

範例3-7

透過原型加上方法，所有物件都可操作

var ary1 = new Array(12, 55, 43, 32, 44);
var ary2 = new Array(12, 33, 1);
Array.prototype.sum = function () {
    var output = 0;
    for (var i = 0; i < this.length; i++) {
        output += this[i];
    }
    return output;
};
console.log(ary1.sum());
console.log(ary2.sum());

範例3-8

如果透過原型家方法，但物件上也有自己的方法，會以物件自己的方法為優先

var ary1 = new Array(12, 55, 43, 32, 44);
var ary2 = new Array(12, 33, 1);
Array.prototype.sum = function() {
    var output = 0;
    for (var i = 0; i < this.length; i++) {
        output += this[i];
    }
    return output;
};
ary1.sum = function () {
    return "nothing";
};
console.log(ary1.sum());
console.log(ary2.sum());

🍭尋找方法或屬性時會優先尋找本身物件，若找不到才尋找原型(__proto＿)上的

物件導向-物件

類別(class),物件(object)，在ES5沒有類別而是透過function提供類似建構函式的用法定義類別。
透過new可由類別建立一個新的物件，可把它想成:
☘ 類別(class) => 模子, 物件(Object) => 成品
透過模子可生產成品，但不會直接拿模子來使用，因為誠品是拿來用的，也不會拿成品去產生新的成品

原型才會有prototype
function Foo();
console.log(typeof Foo.prototype);
物件才會有 ＿proto＿ (物件的原型鏈)
var a new Foo();
console.log(a.prototype);
console.log(a.__proto＿ === Foo.prototype);

範例3-9

類別和物件的用法

Array.push(); //找無
var ary = new Array();
ary.push();
var ary2= new ary();

物件導向-共用方法

原型是針對類別加上共用的方法，已達到節省系統資源目的，避免重複產生方法

範例3-10

使用原型來添加共用的方法，解決範例3-3工廠方式產生的不同物件問題

function CreatePerson(name, gender) {
    this.name = name;
    this.gender = gender;
}
    CreatePerson.prototype.showName = function () {
        console.log("my name:" + this.name);
    };
    CreatePerson.prototype.showGender = function () {
        console.log("my gender:" + this.gender);
    };
var p1 = new CreatePerson("Mike", "male");
var p2 = new CreatePerson("Phil", "female");
console.log(p1.showName === p2.showName);  //true

範例3-11

建立物件後增加屬性

function Obj() {
    this.name = "Mark";
    this.run = function() {
        // code
    };
}
var a = new Obj();
var b = new Obj();
var c = new Obj();
//為每個物件增加屬性
a.height = 15;
b.height = 15;
c.height = 15;
//可以改成這樣
Obj.prototype.height = 15;

🍄因為Obj.prototype === a.proto＿，當新增Obj.prototype.height屬性等同於a.proto＿.height
🍄當a物件找不到height會去proto＿底下尋找height，若找不到在往上層的proto＿一直找到null為止

物件導向-內建物件原型方法

也可替系統內建用原型加方法，達到擴充功能目的

範例3-12

內建物件用原型加方法

String.prototype.toNumber = function () {
    return parseInt(this);
};
var ten = "10";
var fifty = "50";
console.log(ten.toNumber());
console.log(fifty.toNumber());

物件導向-私有函式與變數

透過this才能存取方法和屬性，否則就是私有的函式和變數，只有內部才可存取操作

範例3-13

function CreatePerson(name, gender, year) {
    this.name = name;
    this.gender = gender;
    function caclAge(y) {
        var today = new Date();
        return today.getFullYear() - y;
    }
    var age = caclAge(year);
    this.showAge = function () {
        console.log("my age:" + age);
    };
}
CreatePerson.prototype.showName = function() {
    console.log("my name:" + this.name);
};
CreatePerson.prototype.showGender = function() {
    console.log("my gender:" + this.gender);
};
var p1 = new CreatePerson("Mark", "male", 1985);
var p2 = new CreatePerson("Jack", "male", 1978);
console.log(p1.age);

物件導向-唯讀方法和屬性

設定一個唯獨不可修改的屬性，可透過Object.defineProperty建立屬性

範例3-14

設定唯獨

function CreatePerson() {
    Object.defineProperty(this, "name", {
        value: "Mark",     //值也可以是function
        enumerable: true,  //是否可以被列舉for in
        writable: false,   //是否可以被修改值
        configurable: true  //是否可刪除屬性或修改特的enumerable,writable,configurable屬性
    });
}
CreatePerson.prototype.getName = function() {
    console.log(this.name);
};
var p1 = new CreatePerson();
p1.getName();
p1.name = "John";
p1.getName();

如果定義get,set表示要自行控制屬性存取，就不能再去定義value和writable屬性

var obj = {};
(function () {
    var _name;
    Object.defineProperty(obj, "name", {
        get: function () {
            console.log("讀取_name");
            return _name;
        },
        set: function (value) {
            console.log("寫入_name");
            _name = value;
        }
    });
})();

物件導向-實字模式

使用{…}或[…]種寫法，是透過實字模式(literal notation patterns)產生一個物件或陣列，編譯時JavaScript內部會自動把符號轉換成物件。Json是JavaScript子集合，也是用符號來表示物件或陣列

範例3-15

實字模式

var cat = {};
cat.name = "Mike";
cat.getName = function() {
    return this.name;
};
//也可以這樣寫
var cat = {
    name: "Mike",
    getName: function () {
        return this.name;
    }
};

🐤簡單建立一個物件可以用此方式，但此方式不適合於需建立多個相同物件的方法或屬性
🐤雖然在JavaScript實字模式中的物件屬性名稱可不需引號或使用單引號，但純Json資料格式物件屬性名稱必須一律使用雙引號

constructor:
一般物件導向語言，consrructor是一個方法存在於class中。但在JavaScript的function取代了constructor這個設計，並不需要刻意寫出constructor的method，因為JavaScript在new時就會執行這個function，因此constructor可用來設定物件的屬性(property)或者呼叫方法(method)，初始化這個物件讓之後物件能正確運作
🍬prototype原型本身是個物件，同時具備constructor和＿proto＿，constructor指向建構函式，而＿proto＿則指向」Object.prototype

物件導向-實字方式操作

將實字方式建立的物件轉換成JSON字串

範例3-16

var json = {
    name: "Mike",
    show: function () {
        console.log(this.name);
    },
    empty: undefined,
    nothing: null
};
var jsonString = JSON.stringify(json);
console.log(jsonString);
var jsonString = JSON.parse(jsonString);
//{"name":"Mike","nothing":null}

物件導向-命名空間

若有相同的函式名稱會被覆蓋，但透過命名空間可區分相同的方法名稱

範例3-17

相同名稱的函式會被覆蓋

function getUser() {
    console.log("a");
}
function getUser() {
    console.log("b");
}
getUser(); //b

透過命名空間區分相同的方法名稱

var ns = {};
ns.common = {};
ns.custom = {};
ns.common.getUser = function () {
    console.log("a");
};
ns.custom.getUser = function () {
    console.log("b");
};
ns.common.getUser();  //a
ns.custom.getUser();  //b
